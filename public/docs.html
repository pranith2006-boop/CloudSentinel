Technical Documentation

Integrate CloudSentinel observability and validation into your own CI/CD pipelines.

Getting Started

To begin using the CloudSentinel API, you must first authenticate with an admin account. Once authenticated, you can trigger cloud test suites, fetch real-time cluster metrics, and monitor recent test execution trends programmatically.

API Reference

GET /api/metrics

Fetches global cluster telemetry and aggregated pass/fail rates.

{ "success": true, "data": { "totalRequests": 45, "testPassRate": 92 } }

POST /api/tests/trigger

Launches a new automated cloud validation suite.

{ "status": "PASSED", "runId": "RUN-782", "duration": "450ms" }

GET /api/tests/recent

Retrieves the last 10 test execution results.

[ { "runId": "RUN-782", "status": "PASSED", "coverage": "88%" } ]
Authentication

CloudSentinel uses a mock JWT authentication flow. Send a POST request to /api/auth/login with your credentials to obtain a temporary session token.

CLI Access

You can also interact with the platform using our official CLI tool:

$ cloud-sentinel trigger --env prod
